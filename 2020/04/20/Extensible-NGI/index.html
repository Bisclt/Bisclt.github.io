可扩展的下一代互联网讲座（李丹）

讲座：可扩展的下一代互联网（李丹）

梅特卡夫定律

一、可扩展是互联网的使命

  • 国际互联网协会 (Internet Society, ISOC)

    Unlike the telephone network, which for years in most countries was run by a single company, the global Internet consists of tens of thousands of interconnected networks run by service providers, individual companies, universities, governments, and others.

    The Internet is for everyone.

    open, globally-connected, secure, and trustworthy.

  • “可扩展互联网”的实现手段： “开放” + “互联”

  • 下一代互联网建立在IPv6的极大地址空间上

  • “国际互联”的基本设计原则（可扩展性）

    • 分布式运行：互联网不统一管理

    • 不改变网络的内部通信机制

    • 网络之间的互联互通服务尽量简单：无连接分组交换、尽力而为

  • IP协议：用于“国际互联”的协议（无连接 + 分组交换 + “尽力而为”的转发 + 基于IP地址的寻址），充分”放权”

    • 互联网对内容提供者不做任何限制
    • 互联网的开放模式冲击电信网运营商
    • 互联网不为任何特殊应用服务
    • 互联网并非对应用的需求“视而不见”
    • 互联网对每个网络的底层通信介质不作限制
    • 互联网对每个网络地层通信的“实际“转发协议不限制
    • 互联网对用户开放，“尽力而为”是在有限资源的情况下的社会主义
    • 互联网对流量开放，流量由发动者驱动：开放 vs DoS
    • 互联网对标准和标准的制定者开放，IETF是国际民间组织
      • 芯片设计领域：指令集标准是要付费使用的
      • 电信网：给持有专利的公司付费

  • 针对IP的非议：

    • IP协议没有服务质量保障
    • IP协议没有移动性支持
    • IP协议缺乏安全性考虑，没有支持网络的“内生安全”

  • 有没有比IP协议更好的互联方案？——谁也不知道

  • 互联网 ！= 计算机网络 ——一系列误导性名词：工业互联网、物联网……

二、IPv6协议增加了互联网的可扩展性

  • 从根本上解决困扰互联网“开放”和“互联”的地址空间问题

  • 当前许多行业网络发展的误区：

    • 以业务需求为纲

    • 只关心局部网络需求，没有全局视野，形成“网络孤岛”

  • 地址分配的可聚类性更强

三、其他互联网方案

  • GENI计划（美国）
  • 软件定义网络SDN
    • 核心思想：路由器的控制平面与数据平面分离，把控制平面集中到一个中央控制器上
    • 可扩展性差，因为互联网是分布式的
  • 内容中心网络ICN / 命名数据网络NDN
    • 技术思路：用“内容名字”取代“IP地址”成为路由寻址的核心（Underlay实现）
    • 本质上取代了DNS的功能，把内容寻址实现到路由器上
  • 服务质量驱动的未来互联网设计
    • 强调服务质量控制（与互联网的可扩展性本质相悖）
    • 对 new IP 的批判
